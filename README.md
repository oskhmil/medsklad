# Liki.on! — парсер аналітики

Цей скрипт конвертує Excel-звіт «Відомість по партіях товарів» з 1С у компактний JSON для аналітичного розділу додатку.

## Як запустити

```bash
# один раз встановити залежність
pip install openpyxl

# запустити парсинг
python3 parser/parse.py /path/to/Аналіз_відділень.xlsx data/analytics.json
```

Після успішного виконання оновіть коміт:

```bash
git add data/analytics.json
git commit -m "chore(data): refresh analytics snapshot"
git push origin Work
```

За 1–2 хвилини нові дані будуть доступні усім користувачам додатку.

## Що всередині

- **Вхід:** xlsx (~12 МБ) з 1С — щомісячна оборотно-сальдова відомість з колонками початковий залишок / надходження / видаток / кінцевий залишок (кількість + собівартість).
- **Вихід:** JSON (~4 МБ), gzip'ed до ~900 КБ браузером.
- **Час виконання:** ~30 секунд на ноутбуці.
- **Ієрархія:** зчитується з `alignment.indent` у колонці A:
  - `indent=2` → напрям діяльності (03-НСЗУ, 67-Благодійна, тощо)
  - `indent=4` → група підрозділів
  - `indent=6/8/10` → ланцюжок «Склад/Підрозділ» (до 3 рівнів)
  - `indent=12/14` → документи-операції (пропускаються, агрегуються у підсумках)
  - `indent=0` без тексту в A → рядок товару (колонка B = назва, C = серія)

## Якщо структура файлу зміниться

Парсер оборонний: невпізнані рядки логуються як `unknown_levels`, не ламають виконання. Якщо в нових даних бачите дивні або відсутні розділи — перевірте цей лічильник у stderr-виводі.

Нові напрями, групи, підрозділи додаються автоматично — код не має хардкоду конкретних імен.
